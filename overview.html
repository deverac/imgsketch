imgSketch

imgSketch allows encoding human-editable vector data in a Query String of an 'img' element attribute. E.g. 

    <img class='pgon' data-src='?z='>

The vector data will be used to generate an image which will then be set as the 'img' source.

To enable imgSketch:

  1. include the 'imgsketch-min.js' Javascript file. 
        <script src="imgsketch-min.js"></script>
  2. In the 'body' tag, add 'imgSketch.init()'. 
        <body onload="imgSketch.init();">

Alternately, instead of using 'onload=...', add the following to a '<script>...</script>':

    window.onload = (event) => {
        imgSketch.init();
    };

The 'init()' method takes two, optional parameters. 

    imgSketch.init(className, attributeName);

The first parameter, 'className' is the class name which is used to select the 'img' elements to operate on; the default is 'pgon'. The second parameter 'attributeName' is the attribute name which contains the Query String. If not specified, the 'data-src' attribute name will be searched. If 'data-src' does not exist, the 'src' attribute name will be searched.

If the 'src' attribute contains the Query String, the browser will attempt to fetch an image from the server (which is normal browser behavior). To avoid this, 'data-src', or some other attribute name can be used.

Drawing text is supported with the following restrictions:

  • Rotate y and rotate z have no effect.
  • Semantics of color is reversed: lowercase letter is used to fill text, while uppercase letter is used to outline.
  • Linewidth only applies when capital letter is used for the color.

When specifying the text to be displayed, the only valid characters are:

  • letters (a-z, A-Z)
  • numbers (0-9)
  • tilde (~)
  • dash (-)
  • period (.)
  • underscore (_)

Note that 'space' is not included in the list above. To be a valid Query String, any character that is not in the list above must be encoded as a two-character ASCII code prefixed with a percent ('%') character, however some browsers may allow and correctly interpret non-valid characters.

For convenience, when specifying text, two consecutive tilde characters ('~~`) will be converted to a newline, and each dash ('-`) will be converted to a space.

Query String Parameters

Vector data is defined using the 'z' and 'zN' query string parameters. Other query string parameters can be specified to control imgSketch behavior.

Parameter Z-Field Description
bg c, nnnnnn Background color of image. The color can be a letter from the color dictionary (e.g. 'g') or can be a six-character hex value.
color c, nnnnnn Set default drawing color. The color can be a letter from the color dictionary (e.g. 'g') or can be a six-character hex value.
colord nnnnnn,nnnnnn,... Specify colors for the color dictionary. This allows replacing any number of colors in the color dictionary. e.g. 'colord=,,,aabbcc' will set the 'd' slot to '#aabbcc'. If twenty-six colors are specified, the entire color dictionary will be replaced.
cxz nnnnnn,nnnnnn Specify colors for the 'x', and/or 'z' entry of the color dictionary. The color dictionary contains 24 colors. Two letters, x and z are not used. the 'cxz' parameter provides an easy way to define a color for the x and z slots. e.g. 'cxz=a1111c' will set the 'x' slot to '#a1111c'. 'cxz=,e2222a' will set the 'z' slot to '#a1111c'.
dim n OR 'x,y' Set the image dimensions. (e.g. 'dim=100,200') If a single value is specified, it will be used for both dimensions. If 'dim' is set, it will override the automatically computed value for the image dimensions.
family m,n,s Set the default font-family.
help n Display help. The vector data is ignored. Instead the generated image will be a page of imgSketch help. (See Examples below.)
linewidth a-z Set the default linewidth. 'a'=1, 'b'=2, 'c'=3, etc.
pad n Add padding to each edge of the image. This will increase the image dimensions, but only if 'dim' has not been specified.
palette 1,2 Display the color palette. The vector data is ignored. Instead, the generated image will be a page containing the color dictionary along with the color-letters. If '1' is specified, the full color palette is shown. If '2' is specified, a subset is shown. (See Examples below.) The 'help' parameter will override this parameter.
polysize n Set default size of polygons.
reload n Reload page after n seconds. n can be a float. This behavior can be useful to have the browser automatically reflect changes without having to manually refresh the page.
scaleimg n Scale the entire image by n.
textheight n Set the line height for multi-line text. (e.g. 'textheight=2' will generate double-spaced text)
textsize n Set the default textsize.
z Z-Fields Define a polygon. See the Z-Fields table below.
zN Z-Fields Define a polygon which is drawn in layer N. N must be 1-9. (e.g. z1,z2,...,z9) Polygons drawn in higher layers will overwrite polygons drawn in lower layers.

Z-Fields

Name Values Description
Scale n Scale the polygon.
Shape c,l,t,s,p,h,e,o,x,m Polygon shape to draw. If capitalized, the final edge of the polygon will not be drawn. c=circle, l=line, t=triangle, s=square, p=pentagon, h=hexagon, e=heptagon, o=octagon, x=text, m=multigon.
Color a-z Color to use. If capitalized, the polygon will be filled with the specified color. Letters 'x' and 'z' do not exist in the color dictionary, but can be added with the 'cxz' or 'colord' query string parameter.
Offset 'x,y' Position of the center of the shape. All shapes are created and centered at (0,0).
Linewidth a-z, _ Linewidth to draw the shape represented as a letter. 'a'=1, 'b'=2, 'c'=3, etc. '_'=Default
Size n Size of the polygon or text. Ignored for 'm' (Multi-gon).
Flag _ Literal underscore character.
Rotate 'z,y,x' Angles in degrees. Rotations do not preserve 3-D rotations; the z-value for points are always set to zero. This does not do 3-D rotations.
Family m,n,s,_ Font family to use for text. This is only used for Text ('x'). 'm'=Monospace, 'n'=Sans-serif, 's'=Serif, '_'=Default
Extra Arc OR Text OR XYPoints Additional info, depending on the shape. For a circle, Arc is optional and specifies the start and end in degrees. (e.g. '45,90') For Text, the actual text must be specified. (e.g. 'Hello~~World') For a Multigon, XYPoints specifies the x,y points of the multigon separated by underscores. (e.g. '2,3_20,3_20,30_10,20')

A Z-Field is composed of the following fields, in order: 

  Scale Shape Color Offset Linewidth Size Flag Rotate Family Extra

The fields are concatenated to give the following format: 

  ShapeOffset_Size_Rotate_Extra     The 'Scale' and 'Color' fields are omitted.
                                    The three underscores are the 'Linewidth', 'Flag', 'Family' fields.

Examples: 

     t                A triangle at (0,0) with default size
     t10,20           A triangle at (10,20) with default size
     t10,20_30        A triangle at (10,20) with size 30
     t10,20_30_15     A triangle at (10,20) with size 30 rotated 15 degrees about the z-axis
     t10,20_30_,25    A triangle at (10,20) with size 30 rotated 25 degrees about the y-axis
     t10,20_30_,,35   A triangle at (10,20) with size 30 rotated 35 degrees about the x-axis

To scale the shape, prepend the shape with the scale factor: 

     2t10,20           A triangle at (10,20) with default size scaled by a factor of 2
     0.5t10,20_30      A triangle at (10,20) with size 30 scaled by a factor of 0.5

To color the shape, insert the color value between the 'Shape' field and the 'Offset' field. Capitalizing the color value will fill the shape with the specified color: 

     tr10,20           A triangle at (10,20) that is drawn in red (r)
     tR10,20           A triangle at (10,20) that is filled with the color red (R)

The first underscoore is the Linewidth field. If specified, the value must be a lowercase letter. 'a' corresponds to the linewidth of 1; 'z' corresponds to a linewidth value of 26. 

     t10,20a           A triangle at (10,20) with linewidth 1 (a) and default size
     t10,20e30         A triangle at (10,20) with linewidth 5 (e) and size 30

The second underscoore is the Flag field. It must always be a literal underscore.

The third underscoore is the Family field. It is only relevant for Text and specifies the font-family (Monospace, Serif, Sans-serif).

All fields, except Shape, are optional. However, sometimes fields must be specified in order for subsequent fields to be parsed correctly. 

     t10,20__,,15      A triangle at (10,20) rotated 15 degrees about the x-axis
     t5k_25            A triangle at the (5,0) with linewidth 11 (k) rotated 25 degrees about the z-axis

The 'Extra' field is used to hold additional data for the Circle, Multigon, and Text shapes. For convenience, a dash '-' can be used to skip all remaining fields and begin the 'Extra' field. 

     c10,20___45,90        Draw an arc from 45-90 degrees at (10,20)
     c10,20-45,90          Draw an arc from 45-90 degrees at (10,20). '-' is used to skip to the 'Extra' field.
     m10,20___1,1_9,1_1,9  Draw a multigon at (10,20)
     m10,20-1,1_9,1_1,9    Draw a multigon at (10,20). '-' is used to skip to the 'Extra' field.
     x10,20___Hello        Draw 'Hello' at (10,20)
     x10,20-Hello          Draw 'Hello' at (10,20). '-' is used to skip to the 'Extra' field.

The left edge and bottom edge of the canvas that is drawn upon will be automatically increased in order to display all shapes and text, however the linewidth is not accounted for and will be clipped. You may wish to use pad or dim to enlarge the canvas further to prevent clipping.

If Offset is not specified (or is '0'), the shape will be drawn at (0,0), which is located in the upper-left corner. Positive-x is down. Positive-y is right.

The maximum length of a Query String is browser-dependent. Many browsers allow at least 8,000 characters and may allow 100,000 or more characters.
